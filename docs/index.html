open‑serpapi

An OSS, CLI agent-first, self-hostable alternative to SerpApi. One tiny native binary that turns raw search pages into clean JSON.

GitHub ↗ Changelog →

The whole idea

# fetch is a plugin — any program that prints a page to stdout
<fetcher> "machin programming language" | ./open-serpapi parse --engine google

{ "organic_results": [ { "position": 1, "title": "...", "link": "...",
                        "displayed_link": "...", "snippet": "..." }, ... ] }

SerpApi sells three layers: fetch (get past blocking), parse (messy HTML → structure), and schema (a stable JSON shape). The proxy/CAPTCHA arms race can't be won self-hosted, so open-serpapi owns parse + schema and makes fetch a plugin. The binary never touches the network — it's deterministic and offline-testable.

🔎

Six engines, clean JSON

Google organic, Google Maps, DuckDuckGo, Bing, Startpage and Brave — titles, links, snippets and display links, normalized.

🧩

Bring your own scraper

curl, a headless browser, a paid API, or a cached file — if it prints a page to stdout, it's a fetcher.

🛡️

Past the CAPTCHA wall

An anti-detect browser fetcher (kept separate) gets real results where plain requests get blocked — verified live.

📝

New engines without code

A built-in CSS-selector engine: add a clean-markup engine with a JSON file. DDG, Bing, Startpage and Brave are defined this way.

Quick start

# build the single binary (needs the machin compiler)
./build.sh

# parse saved bytes, fully offline — no fetcher needed
./open-serpapi parse --engine google < fixtures/google/machin-lang.raw.html

# add a search engine = drop a JSON file in engines/, no recompile
<fetcher> "query" | ./open-serpapi parse --engine duckduckgo
